## 🤝 貢献の始め方

### 1. Issue（課題）を確認する

まずは、[GitHubのIssuesページ](https://github.com/Aynyan2828/autonomous_ai_BCNOFNe_system/issues)を見てみましょう。

- **バグ報告**: `bug` ラベルが付いています。
- **機能提案**: `enhancement` ラベルが付いています。
- **ヘルプ募集**: `help wanted` ラベルが付いています。

自分が貢献できそうなIssueがあれば、コメントで「私がやります！」と宣言してください。

### 2. 新しいIssueを作成する

新しいバグを見つけたり、素晴らしいアイデアを思いついた場合は、新しいIssueを作成してください。

- **バグ報告**: バグが再現する手順を詳しく書いてください。
- **機能提案**: なぜその機能が必要なのか、どのように役立つのかを説明してください。

### 3. Pull Request（プルリクエスト）を送る

コードの修正や機能追加を行った場合は、Pull Requestを送ってください。

#### Pull Requestの手順

1.  **リポジトリをフォークする**
    - 自分のGitHubアカウントに、このリポジトリのコピーを作成します。

2.  **ブランチを作成する**
    - `main` ブランチから、作業用のブランチを作成します。
    - ブランチ名は、変更内容がわかるようにしてください。（例: `fix-oled-display-bug`）

    ```bash
    git checkout -b fix-oled-display-bug
    ```

3.  **コードを修正する**
    - 必要な変更を加えます。

4.  **コミットする**
    - 変更内容をコミットします。コミットメッセージは、何を変更したのかがわかるように、具体的に書いてください。

    ```bash
    git commit -m "Fix: OLED画面の表示が更新されないバグを修正"
    ```

5.  **プッシュする**
    - 変更内容を自分のGitHubリポジトリにプッシュします。

    ```bash
    git push origin fix-oled-display-bug
    ```

6.  **Pull Requestを作成する**
    - GitHubのWebサイトから、元のリポジトリ（`Aynyan2828/autonomous_ai_BCNOFNe_system`）に対してPull Requestを作成します。
    - 変更内容や、なぜその変更が必要なのかを詳しく説明してください。
